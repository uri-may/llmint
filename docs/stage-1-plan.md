# LLMint Stage 1: Core Financial Layer

## Context

LLMint is a decentralized, verifiable inference platform — an OpenAI-compatible API where every response is cryptographically signed, payments settle in USDC, and session receipts are anchored on-chain via Arweave. Stage 1 builds the on-chain economics (smart contract) and CLI tooling. No proxy, no attestation signing, no pools yet. The goal is to prove the financial layer works end-to-end: deposit, lock, settle, withdraw.

## Decisions

- **Contract tooling**: Foundry (forge)
- **CLI**: TypeScript, viem, commander
- **Dev chain**: Anvil (local) + Base Sepolia (testnet)
- **Naming**: `llmint` (CLI command), `@llmint/cli` (package), `LLMintVault` (contract)
- **Lock model**: Platform decides lock amount, contract validates `amount <= available`. One active session per wallet.
- **Stale lock recovery**: Time-based auto-unlock (24 hours), callable by anyone.
- **Stablecoin**: USDC (native on Base via CCTP, standard ERC20, 6 decimals). Switched from USDT — better liquidity on Base, no non-standard transfer issues, natural Coinbase ecosystem fit.
- **JWT**: Custom EIP-191 signed token (jose v6 dropped secp256k1). Header `{"alg":"EIP191","typ":"JWT"}`, payload `{sub, iat, exp}`, signature via viem `signMessage`.
- **Repo strategy**: Single monorepo for Stage 1. Will split into two repos later: `llmint` (protocol — contract + CLI, open source) and a separate platform repo (proxy server). Split happens when the proxy is built in Stage 2.

## Smart Contract: `LLMintVault.sol`

**Dependencies**: OpenZeppelin v5 — `SafeERC20`, `Ownable` (platform-only functions), `ReentrancyGuard`.

**State**:
- `IERC20 immutable usdc`
- `mapping(address => uint256) balances` / `locked` / `lockTimestamps` / `sessionNonces`
- `uint256 platformBalance` — accumulated revenue from settlements
- `mapping(address => mapping(uint256 => Settlement)) settlements`
- `Settlement { merkleRoot, chainHash, totalCost, callCount, timestamp, arweaveTxId }`

**Functions**:

| Function | Access | Description |
|---|---|---|
| `deposit(amount)` | anyone | Transfer USDC in, credit balance |
| `withdraw(amount)` | anyone | Withdraw up to `balance - locked` |
| `balanceOf(user)` | view | Total balance |
| `lockedOf(user)` | view | Currently locked |
| `availableOf(user)` | view | `balance - locked` |
| `lockSession(user, amount)` | platform | Lock funds for session. Fails if active lock exists or amount > available |
| `settle(user, totalCost, callCount, merkleRoot, chainHash, arweaveTxId)` | platform | Deduct cost, store settlement, release lock, increment nonce |
| `releaseStaleLock(user)` | anyone | Release lock after 24h timeout. Nonce does NOT increment |
| `withdrawEarnings(amount)` | platform | Withdraw accumulated platform revenue |

**Custom errors**: `InsufficientAvailable`, `ActiveLockExists`, `NoActiveLock`, `LockNotStale`, `SettlementExceedsLock`, `ZeroAmount`, `InsufficientEarnings`

**Note on access control**: Stage 1 uses `Ownable` (single platform address). For production, this needs to change to `AccessControl` with a `PLATFORM_ROLE` (granted to worker addresses for automated lock/settle) and an `ADMIN_ROLE` (multisig for earnings withdrawal and key management). This is a Stage 2+ concern.

**Events**: `Deposited`, `Withdrawn`, `SessionLocked`, `SessionSettled`, `StaleLockReleased`

## MockUSDC

Minimal ERC20 with 6 decimals and public `mint`. Used in Foundry tests and local Anvil deployment.

## CLI Commands

| Command | What it does |
|---|---|
| `llmint init --network <local\|base>` | Create `~/.llmint/config.json` with network presets. `local` = Anvil defaults, `base` = Base mainnet defaults. Individual values overridable via flags. |
| `llmint auth` | Sign JWT with wallet key (EIP-191), print token + expiry |
| `llmint deposit <amount>` | Approve USDC spend + call `deposit()`, print new balance |
| `llmint withdraw <amount>` | Call `withdraw()`, print new balance |
| `llmint balance` | Read `balanceOf` / `lockedOf` / `availableOf`, format as USD |

**Env var**: `LLMINT_PRIVATE_KEY` — wallet private key.

## Contract Test Plan

Foundry tests in `test/LLMintVault.t.sol`:

**Unit tests**: deposit (success, zero, no approval, multiple accumulates), withdraw (full, partial, zero, exceeds, respects lock), lockSession (success, non-platform, active lock, exceeds available, zero), settle (success, non-platform, no lock, exceeds lock, stores all fields, zero cost, increments nonce), releaseStaleLock (after timeout, before timeout, no lock, no nonce increment, callable by anyone), view functions.

**Fuzz tests**: `testFuzz_deposit(uint256)`, `testFuzz_withdraw(uint256, uint256)`

**Invariant tests**: `sum(balances) + platformBalance == usdc.balanceOf(vault)`, `locked[user] <= balances[user]` for all users.

**Scenario tests**: full session lifecycle, withdraw after settle, multiple sessions (nonce tracking), race condition attack (Story 4 from spec).

## CLI Test Plan

Vitest. Unit tests for jwt (sign/verify/expired/tampered), config (save/load/missing), format (USDC parsing). Integration tests against Anvil for deposit, withdraw, balance commands. Shared `setup.ts` starts Anvil, deploys contracts, mints tokens, tears down.

## Deploy Script

## Risks

| Risk | Mitigation |
|---|---|
| Token interface safety | Using `SafeERC20` for safe transfers regardless of ERC20 compliance variations |
| `jose` dropped secp256k1 | Custom JWT with EIP-191 via viem — no external JWT lib needed |
| ABI drift between contract and CLI | Single `build:contracts` script regenerates ABI. CI enforces freshness |
| Foundry + pnpm symlink conflicts | Keep contracts outside pnpm workspace |
